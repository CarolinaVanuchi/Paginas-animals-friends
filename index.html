Descrição Geral

Nesta atividade serão empregados os conhecimentos adquiridos nos seguintes componentes curriculares: Engenharia de Software II, Desenvolvimento Avançado de Software e Desenvolvimento para Dispositivos Móveis. O seu objetivo é integrar os conhecimentos e aplicá-los em um problema único, nesse caso o aplicativo Animal Friends. É importante destacar que a atividade será utilizada para compor os Instrumentos Avaliativos de cada componente e os conhecimentos serão avaliados de forma independente, conforme o Item 5, que trata da Avaliação

Desenvolvimento Avançado de Software

“Utilizar um WebService é uma das maneiras mais comuns de se integrar aplicações diferentes ”. Dessa forma, esta atividade terá como objetivo implementar um serviço web que deverá disponibilizar métodos para receber dados e armazenar em seu banco de dados.

As seguintes especificações deverão ser seguidas:
● Implementar utilizando no mínimo as seguintes ferramentas e tecnologias: IDE Eclipse, Maven, REST com a API JAX-RS, entre outras que se fizerem necessárias.
● O serviço web deverá ter métodos para receber dados e armazená-los no banco de dados. Os dados que serão armazenados são referentes ao maus tratos e animais para adoção.
● O serviço web deverá ter também um método para retornar uma lista das denuncias de maus tratos e animais para adoção. Os dados deverão ser retornados no formato JSON.

Desenvolvimento para Dispositivos Móveis

Neste componente curricular, deverão ser implementados os módulos de maus tratos e de adoção.

Módulo de Adoção

Este módulo deverá ter no mínimo as seguintes as seguintes informações no cadastro: nome, cpf do anunciante, descrição do animal, nome do anunciante, informações de contato, espécie, sexo, porte com o peso, idade, pelagem, raça, castrado?, link para vídeo, data de cadastro, cidade (deverá ter um cadastro próprio) e outros que achar pertinente.
1. Dados como nome, informações de contato e descrição do animal devem ser obrigatórios.
2. Na tela que exibir todos os animais cadastrados, deverá mostrar o nome, e a descrição do animal.
3. Na tela que apresenta a lista das adoções colocar busca por nome do anunciante, espécie e raça.

Módulo de Maus Tratos

Incluir no mínimo os seguintes atributos no cadastro: descrição do animal, informações de contato, imagens, cidade, altitude, longitude e outros que achar pertinente.

4. Dados como descrição do animal e latitude e longitude, devem ser obrigatórios.
5. Na tela que exibir todos os animais cadastrados, deverá mostrar a descrição do animal, e a cidade.
6. Na tela que lista os maus tratos colocar busca por descrição do animal e informações de contato.
Os dados deverão ser salvos no banco banco de dados do dispositivo e enviados para o WebService.

Engenharia de Software II

Com o objetivo de consolidar as teorias e práticas apresentadas na disciplina Engenharia de Software II, o desenvolvimento deste projeto deverá satisfazer às seguintes exigências:.

Controle de Versão do Projeto com Git.

Cada aluno deverá criar um repositório para gerenciar todos os artefatos produzidos neste trabalho. Neste item serão avaliados: organização do repositório, realização de commits periódicos e uso de mensagens significativas para cada commit. O repositório criado deve estar disponível para os professores das disciplinas envolvidas. Recomenda-se o uso do Bitbucket (https://bitbucket.org/) ou do Github (https://github.com/) como repositório central.

Implementação de Testes Automatizados

Utilizar as técnicas e ferramentas estudados em Engenharia de Software II para realizar a implementação de testes automatizados para o servidor de Web Services. Devem ser testados, pelo menos, os objetos de acesso à dados (DAOs) e os métodos utilitários (e.g. validação de CPF). Os testes devem abranger às seguintes verificações:

○ Os dados estão sendo persistidos corretamente?
○ Os resultados das consultas estão corretos?
○ Os métodos utilitários apresentam os resultados esperados?
